An explanation of the "recursive self-improvement loop" prompting pattern applied to marketing tasks with Claude. The core technique inverts the typical one-shot prompt workflow: instead of generating output and shipping it, the prompt instructs Claude to generate, score its own output against explicit criteria, diagnose weaknesses, rewrite, and repeat until it meets the target bar. The author frames this as treating Claude as an autonomous quality loop rather than a text generator.

Practical application: the author uses scoring rubrics (e.g., 10 criteria for image ad concepts, headline strength metrics) embedded directly in the prompt so Claude has a concrete definition of "good" to evaluate against. The loop typically runs 2–4 iterations before the output stabilizes at a quality level that matches or exceeds what a human review cycle would produce. The author credits @maxwellfinn's image ad skill as a concrete example of this pattern in practice. The broader principle is that explicit evaluation criteria embedded in the prompt transform a language model into something closer to an optimization process.

## Key Takeaways

- Recursive self-improvement pattern: generate → score against criteria → diagnose weaknesses → rewrite → repeat until threshold met
- Embed explicit, quantified scoring rubrics in the prompt so Claude has an objective quality bar to evaluate against
- 2–4 loop iterations typically sufficient before output stabilizes
- Particularly effective for marketing copy, ad concepts, and any task with well-defined quality criteria
- Transforms Claude from a one-shot generator into an autonomous quality-optimization loop without human intervention between passes
