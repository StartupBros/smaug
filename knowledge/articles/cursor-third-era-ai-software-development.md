Cursor co-founder Michael Truell outlines three distinct eras of AI-assisted software development. The first era was tab autocomplete — low-entropy, repetitive work automated keystroke by keystroke. The second was synchronous agents, where developers directed prompt-and-response loops while staying in the loop at each step. The third era — now arriving — features cloud agents that run autonomously on virtual machines over hours, returning reviewable artifacts (logs, video recordings, live previews) rather than requiring constant human supervision.

Truell reports that over 35% of PRs merged internally at Cursor are now created by autonomous cloud agents. The developer role is shifting from writing and guiding code to defining problems, reviewing artifacts, and managing fleets of parallel agents. He frames Cursor's new direction as building "the factory that creates software" rather than a code editor. The tab-to-agent transition that took nearly two years is already giving way to the cloud-agent era, which may arrive in under a year.

## Key Takeaways

- Three eras: tab autocomplete → synchronous agents → autonomous cloud agents
- 35%+ of Cursor's internal PRs are now agent-created, running in cloud VMs without human direction
- Cloud agents return artifacts (previews, logs, recordings) enabling parallel review instead of line-by-line oversight
- Developer role shifts from writing code to breaking down problems, setting review criteria, and spinning up multiple parallel agents
- Flaky tests and broken environments — manageable for one human — become critical blockers when every agent run hits them at scale
- The synchronous agent era may last less than a year before cloud agents dominate
