A practical walkthrough of custom Claude Code global commands built specifically for product management work rather than coding. The author manages an Obsidian vault of 4,870+ markdown notes and built three global commands to solve recurring PM friction points: `/today` (surface files modified in the last 24 hours, grouped by vault section with snippets), `/decide` (guided structured decision documentation with context, options, and rationale), and a third command for OKR/quarterly review workflows.

The piece makes an important observation: while AI engineers on X share `/ship` and `/security-review` commands focused on code, PMs are underserved by these patterns. Claude Code can be a productivity layer for knowledge management and decision systems — not just software development. The author shows that the underlying primitives are simple (a `find` command under the hood), but the value is in the structured output, consistent formatting, and reduced friction in high-cognitive-load moments.

## Key Takeaways

- Claude Code commands extend naturally beyond coding into knowledge management, decision capture, and recurring PM workflows
- Global commands that work across projects (not just per-repo) are a significant capability for personal productivity systems
- The `/decide` command pattern — structured capture at the moment of decision — addresses a real PM pain point around decision debt and undocumented rationale
- The Obsidian + Claude Code integration pattern shows how AI agents can navigate and query personal knowledge graphs via file system access
- The "what about everyone else?" framing challenges the engineering-centric defaults of current Claude Code discourse
