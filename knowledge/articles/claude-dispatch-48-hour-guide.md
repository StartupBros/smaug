A detailed 48-hour hands-on test of Claude Dispatch — the mobile orchestrator that lets you spawn and direct multiple Cowork task sessions on your desktop from your phone. Covers setup, real product manager workflows, gotchas with workarounds, and the architectural insight that a persistent knowledge layer (CLAUDE.md + knowledge files in a GitHub repo) compounds across all Claude surfaces.

## Key Takeaways

- Dispatch is not "Claude chat on your phone" — it's an orchestrator. Your phone is the command chair; your desktop does the execution. From one Dispatch conversation you can spawn many parallel Cowork sessions.
- The author ran 60+ task sessions in 48 hours with ~25 minutes of total direction time, with 3+ hours of parallel Claude execution running in the background.
- Main gotchas: folder access must be granted per subtask on desktop, no built-in file transfer (workaround: sync workspace folder to Google Drive), CLAUDE.md should be loaded before delegating tasks so the orchestrator writes precise subtask instructions.
- Decision framework for Claude's 4 remote surfaces: Dispatch (parallel async tasks from phone), Web Sessions (remote coding/prototyping), Channels (event-driven bidirectional via Telegram/Discord), Remote Control (direct computer control, macOS only).
- The knowledge layer insight: store CLAUDE.md and knowledge files in a GitHub repo so every new Claude surface can read it — surfaces change, the knowledge layer compounds.
