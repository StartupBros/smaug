A first-person account of the "Ralph Loop" — an agentic coding workflow where a headless CLI (Claude Code) runs continuously overnight, processing a queue of pre-written PRD tasks while the developer sleeps. The author argues this represents a genuine paradigm shift: developers transition from writing code to becoming Code Product Owners (CPOs) who write requirements, define acceptance criteria, and delegate implementation. In a week-long experiment the author shipped 72 features with 3k unit tests and 2k end-to-end tests — output that would have taken weeks of manual work.

## Key Takeaways

- The Ralph Loop is a for loop running a headless agentic CLI (Claude Code) against a task queue — simple in concept, powerful because of the underlying capabilities that finally made it reliable
- Key enabling technologies: headless lightweight CLI, MCP, agentic browser automation, Agent Skills, long-running agent support, 200k context windows, and reliable tool calling
- Role shift: developers become requirements writers and system architects, not coders — the constraint is now the quality of your PRD and task decomposition, not your typing speed
- Practical tips: don't start from an empty codebase; review your PRD and tasks carefully; run the first few iterations supervised to calibrate your Agent Skills and AGENTS.md
- The "perfect is the enemy of shipped" principle applies — even if AI output isn't as polished as hand-written code, the volume and iteration speed change the economics entirely
