A structured taxonomy of 12 programmatic SEO page patterns, each with URL structure conventions, value requirements, and example search queries. The framework is built around the principle that pSEO should create scalable page systems that solve real user problems—not mass-produce low-value content. The original detailed playbook is from Corey Haines at seoitis.com and the GitHub skill repo.

The 12 patterns: **templates** (downloadable/interactive assets), **curation** (best-of lists with high commercial intent), **conversions** (unit/format conversion tools), **comparisons** (vs/alternative pages), **examples** (inspiration collections), **locations** (service + city pages), **personas** (audience-specific landing pages), **integrations** (tool-pairing pages), **glossary** (definitional/educational pages), **translations** (localized versions), **directory** (categorical tool listings), and **profiles** (person/company pages). Patterns can be layered: locations + personas, curation + locations, integrations + personas, glossary + translations.

## Key Takeaways

- 12 distinct page pattern types, each with different intent signals, URL conventions, and value requirements
- The layering principle (combining patterns) multiplies addressable search volume
- Each pattern has specific value requirements to avoid thin-content penalties—e.g., comparisons need honest analysis with real feature data, not affiliate-biased summaries
- Integrations pages are particularly high-intent because they capture users already committed to a specific tool ecosystem
