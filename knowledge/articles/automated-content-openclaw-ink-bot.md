iuliia shnai (founder of Papermark) built a fully automated content pipeline using an OpenClaw bot called "Ink" that replaced a 3-person writing team. The system processes ~$1–5 per piece of content and runs almost entirely from a phone via Telegram. Ink monitors Reddit and Hacker News for content ideas, checks keyword difficulty and volume via Semrush, decides content format (blog, page, help), writes the article using Papermark's repo as context, takes/uploads screenshots to AWS, commits to GitHub, opens a PR, waits for a Vercel preview to build, and sends a preview link for review. Feedback is given by replying in Telegram, and Ink updates its own instructions over time. The site now hosts 1,270 articles in multiple languages, 400 help and customer stories, and 140 pages — all generated by the bot rather than people.

Key limitations still being worked on: screenshot quality and format reliability, scheduling/cron automation, better cross-repo navigation between the product and marketing repos, and reducing the human review time per article.

## Key Takeaways

- Replaced 3 human content writers with a single OpenClaw bot (cost: ~$1–5/article)
- Full pipeline: keyword research → format decision → writing → screenshots → GitHub PR → Vercel preview → feedback loop
- Entire workflow managed from a phone via Telegram — no desktop required
- Bot self-improves: feedback in chat updates its own content instructions
- Key unsolved pain: screenshot accuracy and cross-repo content awareness
- GSC indexing is fully automated as the final step after merge
