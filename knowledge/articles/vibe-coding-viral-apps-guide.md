A first-person guide from a developer who went from 30 followers to 50M+ impressions by building AI-assisted ("vibe coded") apps publicly. The article covers the full pipeline: finding viral app ideas by consuming content and noting patterns, choosing a lean tool stack (Rork for mobile prototyping, Claude Code for everything else), improving design with reference tools and AI feedback loops, and crafting posts that lead with visual hooks rather than feature lists. The central argument is that virality is repeatable if you optimize for audience relatability, not technical detail.

## Key Takeaways

- Ideas come from pattern-matching viral content, then combining with personal experience to add credibility
- Use 2-3 core tools deeply rather than switching frequently; Rork enables phone-based prototyping without local setup
- Create a shared styles/constants file early to maintain design consistency across screens
- Use ChatGPT to audit screenshots for design issues — useful roughly 60% of the time
- Frame posts around the viewer's emotional experience, not the features you built
- A recognizable series format (e.g., "day X of building Y") creates returning audiences
- Canva mockups with clean screenshots significantly increase scroll-stopping rate
